### 2.1 看行业

#### 2.1.1 汽车行业现状与趋势

| 维度 | 现状描述 | 趋势判断 |
| --- | --- | --- |
| 市场规模 | 中国 ZCU 市场 2024 年规模 **39.3 亿元**，搭载量超 **200 万辆**，渗透率 **8.83%**（佐思汽研） | 2026–2028 年预计翻倍至 20%+，MCU-less 随 ZCU 同步进入爆发窗口 |

| 技术成熟度 | EEA 2.0 → EEA 3.0（ZCU）已是当前主流；EEA 3.x（MCU-less）进入量产导入期 | 2025–2028 为 MCU-less 批量落地关键窗口；EEA 4.0（全软件定义）为概念阶段 |
| 政策环境 | 中国新能源汽车渗透率超 50%；整车轻量化、OTA 升级被纳入多项政策支持方向 | 国产替代（汇顶 GE1101）契合供应链安全政策，国内 OEM 采购意愿强 |

**传统分布式 ECU 的系统性困境**（驱动变革的根因）：

| 矛盾维度 | 具体表现 | 量化数据 |
| --- | --- | --- |
| **硬件冗余** | 每个功能独立 MCU + 外围电路，大量重复 | 整车 ECU 数量 50–100 个 |
| **线束成本** | 点对点布线，线束迂回冗长 | 线束总长 1500–2000m，占整车成本显著比例 |
| **OTA 困难** | 分散 MCU 各有独立固件，需逐一刷写 | 单车型 OTA 节点 50–100 个 |
| **TTM 延长** | 新功能需硬件改版，开发周期长 | 新功能开发周期 12–24 个月 |
| **安全攻击面** | 分散接口缺乏集中防护，入侵路径多 | 攻击面随 ECU 数量线性增长 |

**ZCU 过渡阶段验证**：2020 年前后业界引入"中央计算+区域控制"EEA 3.0 架构，ZCU 汇聚物理区域内的 ECU，通过以太网连接 CCU。但 ZCU 本身仍包含独立 MCU，控制逻辑并未真正集中。MCU-less 是 ZCU 路线的"最后一公里"——将边缘节点 MCU 彻底移除，控制逻辑全部上移至 ZCU/CCU。

**三代 EEA 架构演进对比**：

| 指标 | EEA 2.0 分布式（痛点基准） | EEA 3.0 ZCU（当前主流） | EEA 3.x MCU-less（目标态） |
| --- | --- | --- | --- |
| ECU/节点数 | 50–100 个 | 30–50 个 | 50–80 个 RCP 节点（无 MCU）|
| OTA 节点数 | 50–100 | 30–50 | **仅 4–8 个 ZCU/CCU** |
| 线束总长 | ~2 km | ~1.4 km | **~0.8 km（减 60%）** |
| 边缘固件数量 | 50–100 套 | 30–50 套 | **0（零固件）** |
| 新功能 TTM | 12–24 个月 | 8–15 个月 | **3–6 个月（仅改 ZCU 软件）** |
| 整车 BOM 节省 | 基准 | −15% | **−30%（对比 2.0）** |

### 2.2 看客户

#### 2.2.1 汽车领域客户

**目标用户画像**

    + 用户类型：主流整车 OEM（吉利/长安/理想/小鹏等）及其 Tier1 供应商（安波福/博世/大陆等）
    + 规模：年产量 10 万辆以上的量产平台；2026–2028 新平台立项阶段的架构决策团队
    + 核心使用场景：车身域控制器（门锁/座椅/氛围灯/天窗/雨刮），ASIL-B 以下节点

**核心痛点（Top 3）**

    . 痛点一：整车 OTA 涉及 50–100 个边缘节点固件，维护成本极高，任一节点刷写失败都可能影响交车；固件版本管理混乱导致召回风险持续存在
    . 痛点二：点对点布线导致线束总长 1500–2000m，线束占整车成本显著；新能源车每减重 1kg 可提升续航约 0.3%，但传统架构减重空间极小
    . 痛点三：新功能开发周期 12–24 个月，每次功能迭代需硬件改版 + RTOS/固件重开发 + 功能安全重认证，严重制约 SDV（软件定义汽车）战略落地

**MCU-less 对 SDV 核心诉求的支撑**：

| SDV 核心诉求 | MCU-less 如何支撑 |
| --- | --- |
| OTA 功能迭代 | 边缘零固件 → OTA 复杂度从"整车百节点"降至"ZCU/CCU 数节点" |
| 功能按需订阅 | ZCU 侧软件控制 → 同一硬件平台可动态激活不同功能 |
| 软硬件解耦 | RCP 芯片标准化 → 更换 ZCU 软件不影响边缘硬件，反之亦然 |
| 快速迭代 | 边缘无软件 → 新功能仅需 ZCU 侧软件升级，无需重新认证边缘固件 |

**功能安全适用边界**（客户最关心的红线）：

| 应用域 | MCU-less 适用性 | 原因 |
| --- | --- | --- |
| 车身域（门锁/座椅/照明/雨刮） | ✅ 强烈推荐 | 控制逻辑简单，无本地安全计算需求 |
| 智能电源域 | ✅ 推荐 | 电源路由逻辑可由硬件承担 |
| 热管理系统 | ✅ 可用 | 多执行器协调，逻辑可上移 |
| 底盘气悬域 | ⚠️ 不推荐 | 本地控制算法复杂，域控 CPU +10% 负载 |
| 制动/转向/气囊（ASIL-D） | ❌ 禁止 | 必须本地降级（Fallback），MCU 不可去除 |

> **结论**：MCU-less 的适用范围是**车身舒适域（Body Domain）**，ASIL-B 以下场景。ASIL-D 功能安全系统必须保留本地 MCU 作为降级保障。

#### 2.2.2 机器人领域客户

**目标用户画像**

    + 用户类型：人形机器人整机厂（宇树/优必选/小米机器人/特斯拉机器人等）及关节模组供应商
    + 规模：追求 ¥99,999 量产价格目标的整机厂；外设节点密度 >30 个的商用机型（如 UBTECH Walker X 的 200+ 节点）
    + 核心使用场景：皮肤触觉传感器阵列（32–128 路 IO）、LED 状态灯、散热风扇群、夹爪末端执行器（立即可落地）；关节模组电机控制（2027–2028 拐点后）

**核心痛点（Top 3）**

    . 痛点一：关节模组每颗 MCU + FOC 固件需独立开发维护，30–50 套固件碎片化，OTA 需逐一刷写，新算法（如新增抓取策略）TTM 达 6–12 个月
    . 痛点二：关节控制板 PCB 30×40mm，每个手指关节 MCU + 驱动芯片体积超过关节可用空间（<10mm×15mm），是灵巧手小型化的直接障碍
    . 痛点三：臂部 7 个关节每关节需 8–14 根线缆穿越折弯处，整机线束 1.5–2 kg，每降 100g 可提升末端负载能力约 10%，线束是关键重量瓶颈

**机器人外设节点 MCU-less 机会**（立即可落地，无技术风险）：

| 外设节点类型 | 当前方案 | MCU-less 替代方案 | 收益 |
| --- | --- | --- | --- |
| **皮肤/触觉传感器阵列** | 多个分布式 MCU 采集 + CAN 上传 | RCP 节点直接转发 ADC/GPIO 数据 | PCB 减小 50%+，重量降低 |
| **状态 LED / 显示条** | MCU + SPI 控制 LED 驱动 | RCP 节点直接控制 LED Driver | 零固件，OTA 复杂度归零 |
| **散热风扇/泵** | MCU PWM + 转速反馈 | RCP 节点 PWM + 硬件测速 | 响应更快（硬件状态机 μs 级）|
| **末端执行器（夹爪）** | MCU + 力传感器 + 电机驱动 | 专用夹爪控制 ASIC + RCP 接口 | 夹爪模块标准化互换 |
| **电池管理子模块** | MCU + AFE 芯片 | ASIC + 10BASE-T1S 上报 | 安全可靠性提升 |
| **摄像头电源/使能** | MCU GPIO 控制 | RCP 芯片直接输出 | 节省独立 MCU |

### 2.4 看自己

**✅ 我们的优势**（以国内 Tier1/整机厂视角）

    + 优势 1：本土协同优势——与汇顶 GE1101 等国产芯片处于同一生态圈，联合开发响应速度快于国际 Tier1；联合 POC 验证中（理想汽车/宇树科技等）
    + 优势 2：架构迁移能力——已具备 ZCU SOA 服务化开发经验（参考小鹏 XEEA3.5 的 300+ 原子化服务理念），可直接复用于 MCU-less 控制逻辑上移
    + 优势 3：双赛道覆盖——汽车车身域 + 机器人外设节点底层技术高度共通（100BASE-T1 + RCP + SOA），一套能力体系同时覆盖两个市场，投入产出比极高
    + 优势 4：成本优势——汇顶 GE1101（~14 RMB）相比 ADI AD3304（~$1.3 ≈ 9–10 元）差距已缩小，且 IO 密度（120 路 vs 12 路）优势显著
    + 优势 5：汽车→机器人迁移路径已验证——小米汽车（YU7 线束减少 40%）→ 机器人团队直接复用工程积累；小鹏汽车 XEEA3.5（线束减重 30%）→ 小鹏鹏行机器人沿用 10BASE-T1S 规划

**⚠️ 我们的短板**

    + 短板 1：RCP 中间件能力薄弱——传统嵌入式固件团队尚未向"服务化软件工程师"完成转型；ZCU SOA 服务开发需要全新能力体系
    + 短板 2：功能安全认证路径不清晰——MCU-less 节点 ASIL-B 认证工具链缺失；认证边界从"逐节点认证"→"ZCU 层整体认证"的迁移工程成本高，将成为 OEM 采购的摩擦点
    + 短板 3：机器人专用验证不足——100BASE-T1 在关节动态弯折/振动环境（每天 10,000–100,000 次）下的可靠性尚无专项数据；AEC-Q100 汽车标准无法直接适用
    + 短板 4：10BASE-T1S 工具链不成熟——与 CAN 相比，调试工具链（示波器/协议分析仪）均不完善，客户导入摩擦大

### 2.6 总结

- **结论 1**：MCU-less 技术已完成从概念到量产的跨越（宝马 2024），进入规模化复制阶段，2026–2028 是关键窗口，入场越早技术壁垒越厚。

- **战略意义**：谁先建立起 100BASE-T1 + RCP 中间件 + 功能安全认证（ASIL-B）的完整能力体系，谁就能在 2026–2028 的行业规模化窗口中占据 Tier1 卡位优势，实现从"ECU 模块供应商"到"软件定义硬件解决方案商"的战略升级。

## 3. 技术方案（What）

> 🎁 明确技术目标，设计技术方案，定义关键技术路径与成功指标。

### 3.1 技术方案

#### 3.1.1 总体架构：双重解耦

MCU-less 的核心价值不仅是"去掉一颗 MCU"，而是实现两个层面的解耦：

```
[软硬解耦]  硬件标准化 + 功能软件化 → 支持 OTA 升级而无需硬件改版
[软软解耦]  应用逻辑 ⊥ 通信协议栈 → 微服务化部署，通过标准接口调用边缘 IO
```

**技术本质**：
- 边缘节点仅保留 **MCU-less 芯片（硬件状态机）+ 驱动芯片（HSD/H-Bridge/LED Driver）**
- MCU-less 芯片内部无操作系统、无应用程序，通过硬件电路直接执行 ZCU 下发的 GPIO/PWM/SPI/I2C 指令
- ZCU/CCU 承担所有控制逻辑和状态机，形成完全集中式控制

**传统 ECU → MCU-less 节点的硬件革命**：

```
传统ECU节点：MCU + 晶振 + 复位电路 + 存储器(Flash/EEPROM) + 电源管理 + 通信收发器 + 驱动芯片（HSD/H-Bridge/LED Driver）
                          ↓ MCU-less
MCU-less 节点：RCP 芯片（含 MAC-PHY + 硬件状态机）+ 驱动芯片（HSD/H-Bridge/LED Driver）
```

**量化收益**（节点级）：

| 指标 | 传统 ECU | MCU-less 节点 | 节省幅度 |
| --- | --- | --- | --- |
| 节点 BOM 成本 | $15–25 | $8–12 | **约 40%** |
| PCB 面积 | 25×35 mm | 15×20 mm | **约 51%** |
| 设计复杂度 | MCU 选型 + 固件开发 + RTOS | 仅硬件配置，无固件 | 显著降低 |
| 固件开发工作量 | 每节点需独立固件 | **零固件**（硬件状态机）| 100% 消除 |
| OTA 节点数 | 随 ECU 数量线性增长 | 仅 ZCU/CCU 需 OTA | 减少 80%+ |

**节点级 BOM 成本明细节省**：

| 成本项 | 传统 ECU | MCU-less | 节省 |
| --- | --- | --- | --- |
| MCU 芯片 | $3–8 | $0 | $3–8 |
| 晶振 | $0.3–0.5 | $0（RCP 芯片内置时钟） | $0.3–0.5 |
| Flash/EEPROM | $0.5–1.5 | $0（可选外置 EEPROM） | $0.5–1.5 |
| LDO/电源管理 | $1–3 | $0–0.5（部分集成） | $0.5–3 |
| CAN/LIN 收发器 | $0.5–1.5 | $0（集成或替换） | $0.5–1.5 |
| PCB 面积成本 | $2–5 | $1–2.5 | $1–2.5 |
| **合计 BOM** | **$15–25** | **$8–12** | **~$7–13（约 40%）** |

#### 3.1.2 EEA 演进路径

```
EEA 1.0/2.0  →  EEA 3.0  →  EEA 3.x（MCU-less）→  EEA 4.0（全软件定义）
分布式 ECU       ZCU 区域控制    ZCU+MCU-less节点       中央 AI + MCU-less节点
（100–150 ECU）  （20–30 ZCU）   （边缘纯 IO 芯片）     （概念阶段）
```

#### 3.1.3 ZCU 软件 5 层架构（控制逻辑集中侧）

```
应用层（应用逻辑，如座椅控制状态机）
    ↕
原子服务层（标准化服务接口，对外暴露 API）
    ↕
设备抽象层（屏蔽底层硬件差异）
    ↕
RCP 抽象层（统一 RCP 协议操作接口）
    ↕
GPAN/100BASE-T1 SDK（物理网络驱动）
    ↕
[边缘 RCP 节点：纯硬件执行，无 OS，无固件]
```

这与小鹏 XEEA3.5 的 **300+ 原子化服务**（SOA）理念完全一致，实现了真正的"功能上移、硬件下沉"。

#### 3.1.4 机器人应用同构映射

```
汽车 CCU/HPC         ←→  机器人主控计算机（GPU/NPU SoC）
汽车 ZCU（区域控制）  ←→  机器人肢体控制器（左臂/右臂/左腿/右腿）
汽车 MCU-less 节点   ←→  机器人外设节点 + 关节执行层（ASIC）
```

**机器人传统关节模组 → MCU-less 关节模组**：

```
传统关节模组：
  ┌─────────────────────────────────────────────┐
  │  MCU（Cortex-M4/M7）                         │
  │  + 晶振 + Flash + SRAM                       │  ← 软件层（固件 PID/FOC 算法）
  │  + 编码器接口（QEP/SPI）                      │
  │  + 故障检测逻辑（软件实现）                    │
  ├─────────────────────────────────────────────┤
  │  电机驱动芯片（Gate Driver + 功率 MOSFET/IPM）│  ← 硬件层
  │  + 电流采样（ADC + 运放）                     │
  │  + 绝对编码器（每轴磁编码器）                  │
  ├─────────────────────────────────────────────┤
  │  CAN/EtherCAT 收发器                          │  ← 通信层
  └─────────────────────────────────────────────┘

MCU-less 关节模组：
  ┌─────────────────────────────────────────────┐
  │  专用电机控制 ASIC（硬件 FOC 状态机）          │  ← 硬件替代 MCU 软件
  │  （集成：PWM生成 + 电流控制环 + 编码器接口）    │    无 OS，无固件
  ├─────────────────────────────────────────────┤
  │  电机驱动芯片（Gate Driver + 功率 MOSFET/IPM）│  ← 硬件层（不变）
  │  + 绝对编码器（磁编码器）                      │
  ├─────────────────────────────────────────────┤
  │  10BASE-T1S / EtherCAT PHY（集成 RCP 接口）  │  ← 通信层（标准化）
  └─────────────────────────────────────────────┘
```

| 节省项目 | 节省金额 |

**隐性成本节省（难以直接量化）**：
- **OTA 失败成本归零**：边缘节点无固件，不存在 OTA 刷写失败/砖机场景
- **售后返修降低**：固件 Bug 导致的召回风险消除（边缘节点无软件）
- **开发人员成本**：不再需要为每个 ECU 配置嵌入式固件工程师
- **认证成本**：AUTOSAR BSW 层认证仅需在 ZCU 层进行，不在每个边缘节点重复

**综合价值结论（车载）**：以 20 万元售价车型为例，单车硬件成本节省 476 元（约 0.24%），考虑开发、OTA、售后隐性成本，综合 TCO 节省可达 **800–1,200 元/车**，在中高端量产车型上财务回报明显。

- Plug & Play 能力：RCP 节点上电后自动被 ZCU 发现和初始化（拓扑发现机制）；同一硬件可按需配置不同功能

| **EEA SOA 架构（汽车）** | 整车 SOA 服务总线（SOME/IP·DDS）；Feature Store 功能应用仓库；差量 OTA（仅 ZCU/CCU）；ZCU 5 层软件栈 | 验证中 | 🟡 中 | 已有量产参考（小鹏 XEEA3.5，300+ 原子化服务） |
| **机器人实时性分层控制** | 电流环（ASIC 硬件实现，10–50μs）+ 速度环（100–500μs，需 PLCA 严格验证）+ 位置环（1–4ms，100BASE-T1 可满足）+ 外设控制（10–100ms，完全可行） | 验证中 | 🔴 高 | 电流环和速度环必须在专用 ASIC 中硬件实现；外设节点无实时性顾虑 |

### 3.3 关键指标

**⚡ 性能指标**

    + 总线延迟：100BASE-T1 路线 ~50 μs（汇顶 GE1101）；10BASE-T1S 路线 <1 ms（ADI AD3304）
    + 节点 IO 密度：100BASE-T1 路线 120 路/节点（GE1101）；10BASE-T1S 路线 10–20 路/节点
    + 通信速率：100BASE-T1 = 100 Mbps 全双工；10BASE-T1S = 10 Mbps 半双工；CAN FD Light = 500K–2 Mbps
    + 节点数上限：10BASE-T1S 最多 255 节点（典型 8–30）；100BASE-T1 支持 60 节点
    + 机器人关节位置环延迟：10BASE-T1S + RCP 可满足 1–4ms（250–1000 Hz 外环命令）
    + 机器人外设控制延迟：10–100ms，10BASE-T1S / 100BASE-T1 均完全满足

**🛡️ 质量指标**

    + PCB 面积缩减：汽车单节点 PCB 从 25×35mm → 15×20mm（约 51%）；机器人关节 PCB 从 30×40mm → 15×20mm（约 56%）
    + 固件归零：边缘节点固件数量从 50–100 套 → 0，整体 100% 消除
    + 线束减重目标：整车 30–60%（小米 YU7 实现 40%，小鹏 XEEA3.5 实现 30%）；机器人关节走线减少 70%
    + 接插件减少：机器人整机 80–120 个 → 30–50 个（减少 60%）
    + OTA 节点减少：整车 50–100 个 → 4–8 个（减少 80%+）；机器人 30–50 个 → 4–5 个（减少 85–90%）
    + 功能安全边界：车身域 ASIL-B 以下适用；ASIL-D（制动/转向/气囊）禁用；机器人参照 ISO 13849 PLd/PLe 约束关键关节

**📈 业务指标**

    + 整车硬件 BOM 节省：~476 元/车（GPAN BZCU 座椅项目实测）；综合 TCO 节省 800–1,200 元/车
    + 新功能 TTM 压缩：汽车从 12–24 个月 → 3–6 个月；机器人从 6–12 个月 → 1–3 个月
    + 机器人整机节省：¥2,000–3,900/台（30 关节人形机器人）；灵巧手 MCU 从 12 颗 → 1 颗（↓92%）
    + 续航里程提升：线束减重 10–15 kg → 续航提升 1–2%（新能源车）
    + 机器人末端负载提升：手臂自重每降 100g 可提升末端负载能力约 10%；关节控制板减重 10g/关节 × 30 关节 = 整机减重 300g

### E. 机器人线束改善技术路径

```
路径一：10BASE-T1S Multi-drop 总线替代星型走线

传统：肢体控制器 ──── 关节1（独立线对）
                 ──── 关节2（独立线对）    每关节 2–4 对线
                 ──── 关节3（独立线对）

MCU-less：肢体控制器 ──── 关节1 ── 关节2 ── 关节3   单对总线串联（1 对线）
↳ 臂部 7 个关节：从 7×2=14 对走线减至 1 对总线线对，线缆减少 85%

路径二：PoDL（以太网供电）合并数据与电源线
传统（CAN + 独立电源）：  4 线 → ~4 mm 直径
10BASE-T1S（无 PoDL）：  3 线 → ~3 mm 直径
10BASE-T1S + PoDL：       2 线（T1S H/L，供电叠加） → ~2 mm 直径

路径三：节点数量减少（外设节点合并）
示例：机器人手腕区域原有 3 个 MCU（腕关节控制+触觉传感+LED 状态）
→ MCU-less 后 1 个 RCP 节点统一管理
→ 进入腕部线束由 3×4线=12线 减至 4线（总线+电源）
```

**机器人线束量化改善预估**：

| 指标 | 传统架构 | MCU-less 架构 | 改善幅度 |
| --- | --- | --- | --- |
| 穿越关节线缆数 | 8–14 根/关节 | **2–4 根/关节** | **减少 70%** |
| 整机线束总重量 | 1.5–2 kg（估算） | 0.5–0.8 kg | **减重 50–60%** |
| 弯折节点走线直径 | 3–5 mm | **1.5–2.5 mm** | 可靠性大幅提升 |
| 接插件数量 | 80–120 个 | 30–50 个 | **减少 60%** |

### F. 两大领域最终结论

> **MCU-less 技术在汽车和机器人两大领域代表了一次从"分散固件"到"集中服务"的架构范式转变。车载领域是今天的主战场**，技术可行性已验证，商业逻辑已跑通，2026–2028 年是关键规模化窗口；**机器人领域是明天的最大增量市场**，外设节点已具备立即落地条件，关节模组的完整 MCU-less 化依赖专用 ASIC 的诞生，预计 2027–2028 年迎来拐点。对于 OEM、Tier1、芯片厂而言，**现在布局外设节点的低风险突破口，同时储备关节模组的技术能力，是未来 3–5 年最优的资源配置策略。**
