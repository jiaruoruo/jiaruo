## 一、机器人身上哪些模块需要用MCU？

一台典型人形机器人包含 **8-10 个需要MCU的独立功能模块**，每个模块的实时性、算力、安全要求各不相同。

```
┌─────────────────────────────────────────────────────────────────┐
│                     人形机器人 MCU 分布图                        │
│                                                                   │
│  ┌──────────┐                                                    │
│  │  头部     │ ← ④ 视觉/AI感知处理器（MPU/SoC）                 │
│  │ 传感模块  │ ← ⑤ 声学/麦克风处理（低功耗MCU）                 │
│  └──────────┘                                                    │
│  ┌──────────┐                                                    │
│  │  躯干     │ ← ① 主控（运动规划/RL策略执行）                  │
│  │  主控箱   │ ← ② EtherCAT总线主站（实时网络MCU）              │
│  │          │ ← ⑦ 电源管理/BMS（电源MCU）                       │
│  └──────────┘                                                    │
│  ┌──────────┐                                                    │
│  │  手臂关节 │ ← ③ 关节伺服驱动器MCU（×每关节1颗，约7个/臂）   │
│  └──────────┘                                                    │
│  ┌──────────┐                                                    │
│  │  灵巧手   │ ← ⑥ 灵巧手控制器（主MCU × 1 + 从MCU × 手指数）  │
│  └──────────┘                                                    │
│  ┌──────────┐                                                    │
│  │  腿部关节 │ ← ③ 关节伺服驱动器MCU（×每关节1颗，约6个/腿）   │
│  └──────────┘                                                    │
│  ┌──────────┐                                                    │
│  │  安全域   │ ← ⑧ 功能安全MCU（碰撞检测/急停监控）             │
│  └──────────┘                                                    │
└─────────────────────────────────────────────────────────────────┘
```

**需要MCU的8大功能模块一览：**

| # | 功能模块 | MCU核心任务 | 对MCU的核心需求 |
|---|---------|------------|----------------|
| ① | 主控运动控制器 | RL策略执行、步态规划、全身协调 | 高算力、运行Linux/ROS2 |
| ② | EtherCAT总线主站 | 实时网络调度、<1μs同步 | 硬实时、工业以太网协议栈 |
| ③ | 关节伺服驱动器 | FOC电机控制（三环：电流/速度/位置） | 高频PWM（100kHz+）、低延迟ADC |
| ④ | 视觉/AI感知处理 | 深度图处理、目标检测、SLAM | 高算力NPU/GPU，AI推理 |
| ⑤ | 传感器融合处理 | IMU+雷达+力传感器数据融合 | 边缘AI推理、多传感接口 |
| ⑥ | 灵巧手控制器 | 多指独立力位控制 | 多路PWM+ADC、紧凑封装 |
| ⑦ | 电源管理/BMS | 电池均衡、SOC估算、充放电控制 | 精密ADC、低功耗、安全保护 |
| ⑧ | 功能安全监控 | 碰撞检测、急停、安全状态机 | 锁步CPU、功能安全认证（SIL2/ASIL-D） |

## 二、各模块的主流零部件厂商

### 模块①：主控运动控制器（机器人"大脑+小脑"）

**功能**：运行运动规划算法（RL/MPC），下发关节指令到EtherCAT总线

| 类别 | 厂商 | 产品/方案 |
|-----|------|---------|
| **AI算力平台** | 英伟达（NVIDIA） | Jetson AGX Orin（275 TOPS），业界最主流 |
| **AI算力平台** | 瑞萨 | RZ/V2H（8TOPS DRP-AI + 4×A55），国内备选 |
| **AI算力平台** | AMD | Kria KV260 等 FPGA+ARM 方案 |
| **整机系统集成** | 宇树（Unitree） | 自研主控+Jetson AGX |
| **整机系统集成** | 特斯拉 Optimus | 自研AI芯片（FSD Chip） |
| **整机系统集成** | 小米（CyberOne） | 自研控制器 + Cybergear |
| **整机系统集成** | 智元（Agibot） | 自研主控平台 |

### 模块②：EtherCAT总线主站/通信控制器

**功能**：管理全机关节的实时总线通信，周期1ms以内同步所有伺服节点

| 类别 | 厂商 | 产品/方案 |
|-----|------|---------|
| **工业总线芯片** | 瑞萨 | RZ/T2H（内置EtherCAT主站IP） |
| **工业总线芯片** | Beckhoff | AX5000系列（工业标准EtherCAT主站） |
| **工业总线方案** | TI | AM243x（多协议工业以太网SoC） |
| **工业总线方案** | 英特尔/x86工控机 | 运行EtherCAT主站软件（如IGH、SOEM） |
| **通信网关芯片** | HMS Networks | Anybus系列（多协议转换） |
| **新兴方案** | 千讯电子（GPAN） | 无MCU分布式通信方案（取代传统总线架构） |

### 模块③：关节伺服驱动器（机器人最大MCU用量模块）

**功能**：单关节电机的FOC控制，直接驱动BLDC/PMSM电机，每台人形机器人约20-30颗

> 这是MCU需求量最大、竞争最激烈的模块

**主流关节执行器厂商及选型：**

| 整机/执行器厂商 | 产品 | 已知MCU方案 | 来源 |
|--------------|-----|-----------|------|
| **宇树（Unitree）** | B2、H1关节电机（准直驱） | 自研驱动板 | 知识库 |
| **小米** | Cybergear（力矩密度0.29 Nm/g） | 自研 | 知识库 |
| **日本安川电机** | Σ-7 伺服 | 自研DSP/MCU | 行业 |
| **台达电子** | A3/B3 伺服 | 自研，含ST/TI MCU | 行业 |
| **汇川技术** | SV660 伺服（国产主流） | 国产MCU + TI | 行业 |
| **埃斯顿** | 自研伺服 | 国产MCU | 行业 |
| **睿尔曼机器人** | 协作臂关节驱动 | 瑞萨 RZ/T2L | 知识库 |
| **大艾机器人** | 外骨骼关节 | ST STM32 | 行业 |

### 模块④：视觉/AI感知处理器

**功能**：处理摄像头/深度相机/雷达数据，运行目标检测、SLAM、动作识别

| 厂商 | 产品 | 定位 |
|-----|------|------|
| **英伟达** | Jetson Orin NX/AGX | 业界标准，高算力，功耗较高 |
| **瑞芯微（RK）** | RK3588 | 国产替代，6TOPS NPU，成本低 |
| **地平线** | J6 系列 | 国产车规级AI芯片，延伸机器人 |
| **寒武纪** | MLU370 边缘计算卡 | 国产高算力推理 |
| **瑞萨** | RZ/V2H | 8TOPS DRP-AI + Linux，机器人大脑 |

### 模块⑤：传感器融合处理器（边缘AI MCU）

**功能**：融合IMU、力传感器、雷达、皮肤传感器数据，生成环境感知结果

| 厂商 | 产品 | 定位 |
|-----|------|------|
| **英飞凌** | PSOC Edge（集成NPU） | 雷达+传感器边缘AI推理 |
| **意法半导体（ST）** | STM32U5/H5（TrustZone+低功耗） | 传感器MCU主流方案 |
| **TI** | MSPM0/MSP432 系列 | 低功耗传感器MCU |
| **Nordic** | nRF5340 | 无线传感器节点 |
| **博世（Bosch）** | BMI088 IMU（含处理器） | 机器人IMU标准选型 |

### 模块⑥：灵巧手控制器

**功能**：多指独立力位控制，每指1-3个自由度，全手最多24 DOF

**主流灵巧手厂商及选型：**

| 灵巧手厂商 | 产品 | MCU方案 | 来源 |
|-----------|-----|---------|------|
| **因时机器人** | TEF 系列灵巧手 | 自研控制板 | 行业 |
| **灵动科技** | 多指灵巧手 | 自研 | 行业 |
| **Shadow Robot（英）** | Dexterous Hand | FPGA + 自研 | 行业 |
| **特斯拉 Optimus** | 11 DOF灵巧手 | 自研（基于FSD架构） | 知识库 |
| **参考设计（瑞萨方案）** | — | **RA8T2（主）+ RA6/RA4（从）** | 知识库 |
| **参考设计（英飞凌方案）** | — | **PSOC C3 × 多颗** | 知识库 |

**灵巧手典型MCU架构（以瑞萨为例）**：
```
灵巧手掌中枢（主MCU）：RA8T2
  ├─ EtherCAT 从站 → 接入机器人总线
  ├─ CAN-FD 主站 → 下连各手指
  └─ micro-ROS 接口

手指执行器（从MCU）：RA6 / RA4
  ├─ FOC 控制（微电机）
  ├─ 力矩/位置传感
  └─ CAN-FD 从站
```

### 模块⑦：电源管理与BMS

**功能**：锂电池电芯均衡、SOC/SOH估算、充放电保护、多路DC-DC转换

| 厂商 | 产品 | 定位 |
|-----|------|------|

**功能**：独立于主控的安全监控层，负责碰撞急停、关节力矩超限保护（STO功能）

**国产MCU的主要挑战**：
1. **高频PWM与电机专用外设**不足（缺少RA8T2/PSOC C3级别的电机加速单元）
2. **工业以太网协议栈**（EtherCAT从站IP）几乎空白
3. **功能安全认证**（IEC 61508 SIL2+）国内厂商尚无成熟产品
4. **生态和开发工具**与ST/TI差距仍大

## 四、综合结论：MCU选型决策逻辑

```
机器人开发者选MCU的决策路径：

Q1. 是关节伺服驱动器还是其他模块？
  → 是关节伺服：
     优先考虑 瑞萨RA8T2/RZ/T2L 或 英飞凌PSOC C3
     成本敏感：ST STM32G4/H7 也可满足基本需求
  → 是灵巧手：
     推荐 瑞萨RA8T2（主）+RA4（从） 或 英飞凌PSOC C3
  → 是视觉/AI处理：
     优先 NVIDIA Jetson / 瑞萨RZ/V2H / 国产地平线J6
  → 是功能安全：
     必须用 英飞凌AURIX 或 瑞萨RZ/T2M+RX72N 或 NXP S32K
  → 是BMS/电源：
     TI BQ系列 / 英飞凌PMIC / 国产中颖

Q2. 有没有EtherCAT需求？
  → 有 → 瑞萨（最完整EtherCAT生态）/ ST（第三方EtherCAT IP）
  → 没有（用CAN/RS485）→ ST/TI/国产MCU均可

Q3. 有没有GaN功率器件配套需求？
  → 有 → 英飞凌（CoolGaN + PSOC C3，最佳配套）
  → 没有 → Si方案，ST/TI/瑞萨均竞争

Q4. 国产化要求？
  → 强要求国产 → 兆易创新GD32（STM32兼容）是当前最可行路径
                  AI推理用地平线/寒武纪
  → 可用进口 → 按技术性能和生态选型
```
